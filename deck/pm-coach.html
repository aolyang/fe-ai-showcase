pm-coach · Claude Plugin

用 AI 插件，
把 PM 流程跑顺

spec → plan → implement 的 AI 辅助实践。本地文件做主线，人在关键节点卡口。

/pm-init /pm-spec /pm-plan /pm-related
/plugin install pm-coach@infra-fe-ai-resources

Mindset · 核心心智

本地文件优先，GitLab 只发布

CANONICAL

本地 SPEC / plan

是真相源头。反复跑只处理新增内容，不重写。

DERIVED

GitLab issue / milestone

只是发布地。push 同步过去，可被本地覆盖。

需求 issue 在 spec_project，代码 + 本地文件在 code_project。

mindset

Step · /pm-init

跑一次，写入 CLAUDE.md

  • ✓ 校验 glab、选语言
  • ✓ 确认两个项目
  • ✓ 建标签、分角色
  • ✓ 写入 CLAUDE.md

幂等，可重跑。

## pm-coach config
code_project:  klmobile/web
spec_project:  infra-fe/klmobile-specs
labels:  [AI:In Progress, AI:Done, AI:Human Review]
/pm-init

Step · /pm-spec

explore 摸清全貌，push 发 SPEC

explore 做什么

读评论、关联 issue、相关 commit diff，交 issue-explore agent 汇总成本地 SPEC。

/pm-spec explore 100

留下什么

specs/100-sso.md（Summary + SPEC），push 发 GitLab issue。

反复跑只处理新增。

/pm-spec

Step · /pm-plan

make → push → claim

1
make

生成本地 plan

读 SPEC、问澄清，拆成可领任务。

2
push

建 task issue

每任务一个 issue，挂 feature 下。

3
claim

派 agent 实现

建分支、写码跑测试、产 draft MR。

/pm-plan push @plans/100-sso.md  →  task #150/151/152

/pm-plan

Support · 三个支撑件

让流程更稳的辅助能力

  • /pm-related · 跨项目依赖

    登记依赖仓库，扫到别的 repo 的 commit。 /pm-related add ../auth-clone

  • issue-explore agent · 探索主力

    递归关联 issue、读 commit diff、汇总成 Summary。

  • Figma 设计探索

    带 Figma 时分析组件 / token 复用 → .design.md。
